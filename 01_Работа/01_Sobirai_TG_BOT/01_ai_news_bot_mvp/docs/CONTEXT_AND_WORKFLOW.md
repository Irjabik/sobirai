# Sobirai: контекст проекта и как с ним работать

Краткая памятка для себя и для ассистента в Cursor: что за бот, где живёт код, куда коммитить, как выкатывать на Bothost.

---

## Что это за бот

**Sobirai** — Telegram-бот (MVP), который собирает посты из фиксированного списка русскоязычных каналов про ИИ и раздаёт подписчикам:

- режим **мгновенной** доставки (`instant`) или **дайджест** с настраиваемым интервалом (1–168 часов, в интерфейсе — часы и дни до 7 суток);
- фильтры: категории, каналы, mute, пауза;
- дайджест в виде одного сообщения-списка (до 10 пунктов после дедупликации);
- окно свежести для instant: посты не старше **12 часов** в выборке доставки.

Стек: **Python**, **aiogram 3.x** (команды, меню, FSM), **Telethon** (коллектор из каналов), **SQLite** (`DATABASE_PATH`).

## Два репозитория GitHub (важно)

| Назначение | Репозиторий | Что делать |
|------------|-------------|------------|
| **Монорепозиторий** с исходниками в общей структуре папок | [Irjabik/My-project](https://github.com/Irjabik/My-project) | Путь к боту: `01_Работа/01_Телеграмм_бот/01_ai_news_bot_mvp/`. Удобно для разработки вместе с заметками и другими проектами. |
| **Отдельный репозиторий для деплоя** (Bothost и т.п.) | [Irjabik/sobirai](https://github.com/Irjabik/sobirai) | Сюда должен попадать **тот же код приложения** (`app/`, `scripts/`, `requirements.txt`, …). Хостинг обычно указывает **именно этот URL**. |

Коммиты **только в My-project** не обновляют бота на сервере, пока изменения **не перенесены в `sobirai` и не запушены**, затем на панели хостинга не сделан pull/redeploy и перезапуск.

Практика синхронизации (вручную):

1. После правок в `01_ai_news_bot_mvp/` закоммитить в **My-project** (по желанию).
2. Скопировать изменённые файлы в локальный клон **sobirai** (или `rsync` каталога `app/` и нужных `scripts/`), затем `git commit` + `git push` в **sobirai**.

Локальный клон `sobirai` внутри корня My-project может быть в `.gitignore` (`/sobirai/`), чтобы не путать два репозитория.

## Структура каталога MVP

- `app/main.py` — точка входа, `Dispatcher` + **MemoryStorage** (FSM для «свой интервал», ввод `@username` канала).
- `app/bot_handlers.py` — команды, callback-запросы, FSM, вызов логики доставки/настроек.
- `app/keyboards.py` — reply-меню (Режимы, Дайджест, Фильтры, Источники и помощь) и inline-клавиатуры.
- `app/db.py` — SQLite, пользователи, `started_at`, `delivery_events`, дайджесты.
- `app/collector.py` — Telethon, нормализация постов (в т.ч. `media_group_id` для альбомов).
- `app/delivery.py`, `app/service.py` — циклы коллектора и дайджеста.
- `app/formatting.py` — текст дайджеста, подписи интервалов (`format_digest_interval_ru` и т.д.).
- `app/sources.py` — список каналов MVP.
- `scripts/skip_delivery_backlog.py` — **перед запуском после долгого простоя**: сдвинуть `started_at` и `last_digest_sent_at`, чтобы не вывалить накопленные посты (см. docstring в скрипте).

## Поведение для пользователя

- После **`/start`** показывается **reply-клавиатура** из четырёх кнопок; команды из `/help` остаются.
- **`/health`** в тексте `/help` не рекламируется, обработчик есть (операционная команда).
- Дайджест по расписанию **не меняется сам** при обновлении бота: пользователь выбирает интервал кнопками или `/digest <часы>`.

## Запуск и окружение

- Зависимости: `requirements.txt` (`pip install -r requirements.txt`).
- Секреты: `.env` по образцу `.env.example` (**не коммитить**). На Bothost — переменные окружения или загружаемый `.env`.
- Локально из корня MVP: `python -m app.main` (см. README).
- Частый запуск на хостинге: `python -m app.main` или `python3 -m app.main`.

Переменные: `BOT_TOKEN`, `TELEGRAM_API_ID`, `TELEGRAM_API_HASH`, опционально `DATABASE_PATH`, `TELETHON_SESSION_STRING`, `COLLECTOR_POLL_SECONDS`, `DIGEST_POLL_SECONDS`, `LOG_LEVEL`.

## Деплой (Bothost)

1. Git URL: **https://github.com/Irjabik/sobirai.git**, ветка **`main`**.
2. После пуша в **sobirai** — обновить приложение (pull / redeploy) и **перезапустить** процесс.
3. При длительном даунтайме перед первым запуском рассмотреть:  
   `python scripts/skip_delivery_backlog.py` (сначала `--dry-run`) на **той же БД**, что у прод-бота.
