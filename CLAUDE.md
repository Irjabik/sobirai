## 4. ПРАВИЛА РАБОТЫ
Ключевые принципы:
- Никакой лени: ищи корневые причины, никаких временных заплаток.
- Перед началом работы уточняй: зачем это нужно и для чего будет использоваться результат.

Доведение до результата (ПРИОРИТЕТ):
- Всегда доделывай задачу до конца. Без "заглушек" и предложений доделать самому.
- Не пиши "TODO", "добавь сюда", "допиши позже" - сделай сам, полностью, сейчас.
- Если чего-то не хватает для завершения - спроси, но не бросай на полпути.

Управление задачами:
- Для нетривиальных задач (3+ шагов) сначала составь план через встроенный планировщик Claude Code (TodoWrite) и согласуй его перед реализацией.
- Отслеживай прогресс, отмечай выполненное; краткое резюме на каждом этапе.
- Документируй результаты; после исправлений и разборов ошибок обновляй файл `tasks/lessons.md` (создай при необходимости).

Оркестрация процесса:
- Для любой нетривиальной задачи (3+ шагов) - режим планирования; если что-то идет не так - остановись и перепланируй.
- Задачу не считай выполненной, пока не доказал, что работает.
- Для нетривиальных изменений - ищи элегантный способ; костыли переделывай.
- Баг-репорт - исправляй сам: логи, тесты; без лишних вопросов.

Структура ответов:
1. Сначала СУТЬ (решение).
2. Потом ДЕТАЛИ (как это сделать).
3. В конце ВАРИАНТЫ (если есть выбор).

Работа с файлами:
- Никогда не удаляй содержимое файлов без спроса.
- Всегда обновляй README.md при создании новых папок.
- Нумерация: 01_Название, 02_Название - логическая цепочка.
- Названия на русском, четко отражают суть.
- Если для нового файла или папки непонятен путь или целевая директория (несколько вариантов, нет явной структуры, пользователь не указал куда класть) - создавай все в `00_Inbox/` в корне рабочей области; не разбрасывай по случайным каталогам. При необходимости позже перенеси в нужное место.
